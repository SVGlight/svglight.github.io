Welcome to Lesson 10! Async/Await is a modern JavaScript feature that simplifies working with asynchronous operations, such as fetching data from APIs, reading files, or performing timers. It is built on top of Promises but provides cleaner, more readable syntax.

Before Async/Await, chaining multiple Promises with .then() and .catch() could become unwieldy:

fetch("https://api.example.com/data")
  .then(response => response.json())
  .then(data => {
    return fetch(`https://api.example.com/details/${data.id}`);
  })
  .then(details => details.json())
  .then(finalData => console.log(finalData))
  .catch(error => console.error(error));

Using Async/Await, this becomes much cleaner:

async function fetchData() {
  try {
    const response = await fetch("https://api.example.com/data");
    const data = await response.json();

    const detailsResponse = await fetch(`https://api.example.com/details/${data.id}`);
    const details = await detailsResponse.json();

    console.log(details);
  } catch (error) {
    console.error(error);
  }
}

fetchData();

Key points about Async/Await:

  • An async function always returns a Promise.
  • await pauses the execution of the async function until the Promise resolves.
  • Errors can be handled using try/catch.

You can also run multiple asynchronous operations in parallel using Promise.all() with Async/Await:

async function loadAllData() {
  try {
    const [usersResponse, postsResponse] = await Promise.all([
      fetch("/users"),
      fetch("/posts")
    ]);

    const users = await usersResponse.json();
    const posts = await postsResponse.json();

    console.log(users, posts);
  } catch (error) {
    console.error(error);
  }
}

Async/Await allows you to write asynchronous code that reads like synchronous code, making debugging and maintenance easier. Combined with API requests, it’s one of the most important tools for modern web development.

Practice exercises include fetching data from a public API and displaying it dynamically, creating multiple async operations that run in sequence and parallel, and handling errors gracefully. These exercises will prepare you for real-world web applications that depend on asynchronous data.

By the end of this lesson, you should be able to:

  • Understand the syntax and purpose of Async/Await
  • Use Async/Await to handle Promises
  • Handle errors in asynchronous functions
  • Run asynchronous operations in sequence and in parallel
  • Apply Async/Await to API requests and dynamic data handling
